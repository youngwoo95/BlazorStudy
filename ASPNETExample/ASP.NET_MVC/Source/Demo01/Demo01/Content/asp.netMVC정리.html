MVC 패턴 모델 : 사용자가 작업할 데이터를 담거나 표현하는 객체 - 뷰 모델 : 뷰와 컨트롤간에 전달되는 데이터만을 담고 있는 모델 - 도메인 모델 : 업무에 필요한 실질적인 데이터를 정의 해놓은 모델 이 모델은 대부분 저장소에 저장되어 장기간 유지가 된다. 일반적으로 DBMS에 저장한다. 뷰 : 사용자에게 보여지는 인터페이스이다. 보여질 때는 렌더링을 통해 보여지게 된다. 컨트롤러 : 요청(Request)을 받아서 모델을 이용하여 작업처리를 한 후 렌더링 될 뷰를 선택한다. ASP.NET MVC 전체적인 작동 흐름 - 컨트롤러는 System.Web.Mvc.Controller 클래스에서 파생된 C#클래스이다. - 이 컨트롤러에서는 public 메서드들을 액션 메서드라고 한다. - 이 메서드는 라우팅 시스템을 통해서 URL과 연결된다. - 특정 액션 메서드와 연결된 URL 요청이 들어오면 액션 메서드가 동작을 하면서 즉, 컨트롤러가 동작하면서 도메인 모델을 대상으로 필요한 작업을 수행하게된다. - 마지막으로 사용자에게 보여줄 뷰를 선택한다. 이때 필요한 데이터들을 전달하기도 한다. - ASP.NET MVC 프레임워크에서는 이 뷰를 렌더링해서 브라우저에게 응답을 하게되는데 렌더링 할 때 뷰엔진을 이용한다. MVC5에서 사용하는 엔진은 바로 Razor이다. - 초기에는 ASP.NET 엔진을 이용하였고, MVCX3에서 새로운 문법이 적용된 Razor가 도입 - MVC4에서 새로운기능이 추가되어 MVC5까지 이어지고 있다.
